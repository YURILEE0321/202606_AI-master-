### 핵심 사용자 시나리오

**시나리오 1 : AI 탐색 기능 - 단순 문의 응대 챗봇**

- ID : [SC-001]
- 상황 : 단순 기능에 대한 질문
- 목표 : 기능에 대해 변경 사항 업데이트 되어 최신 기능 기준으로 대답해 줄 수 있어야함 [사용자가 달성하고자 하는 것]
- 사전 조건 : 기존 기능에 대한 메뉴얼, 스토리보드, 관련 정리 자료 구축, 변경사항에 대한 릴리즈 노트 데이터, 기능 변경에 대한 공지 메시지에 대한 자료 [시나리오 시작 전 필요한 상태]
- 상세 흐름

| 단계 | 사용자 행동 | 시스템 동작 | 비고 |
| --- | --- | --- | --- |
| 1 | 사용 기능에 대한 단순 질문 | 1) 사전에 정의된 기능에 대한 내용 1차 search
2) 릴리즈노트, 기능 변경에 대한 공지 2차 search
3) 내용 취합하여 답변 제공 |  |
| 2 | 질문에 대한 응답을 찾지 못한 경우 | 1) 답변을 찾을 수 없을 경우 담당자 확인 필요 답변 제공 |  |
| 3 | 질문에 대한 기능이 없을 경우 | 1) 기능 개발 요청할 수 있는 url연결 |  |
- 입력 예시

데이터 적재시 보내는 데이터 유형 변경 방법이 어떻게 되나요?

- 기대 출력

Settings > contiguration 아래 항목 변경이 필요합니다.

- 성공 기준
    - 기준 1: 답에 근접하는 답변 제시
    - 기준 2: 해당 답변을 찾을 수 없을경우 담당자 확인 필요하다는 안내 제시

**시나리오 2 : AI 탐색 기능  - 데이터 유입 관련 문의 대응**

- ID : [SC-002]
- 상황 : 데이터 유입이 안되는 현상에 대한 질문 [언제, 어떤 상황에서 사용하는가]
- 목표 : 문제 원인 파악에 대한 시간 공수 단축 [사용자가 달성하고자 하는 것]
- 사전 조건 : 데이터에 대한 DB 스키마에 대한 정의 구축, 데이터 유입시 진행되는 플로우 자료, 로그 메시지 및 해당 알림에 대한 정의 자료 [시나리오 시작 전 필요한 상태]
- 상세 흐름

| 단계 | 사용자 행동 | 시스템 동작 | 비고 |
| --- | --- | --- | --- |
| 1 | 데이터 유입이 안되는 현상에 대한 문의 | 1. 로그 메시지 탐색
2. DB에서 확인해야할 값 확인 
3. 데이터 유입에 문제 발생에 대한 답변 제시
4. 해당 경로로 확인할 수 없을 경우 추가 확인 안내 메시지 제공 |  |
| 2 |  |  |  |
| 3 |  |  |  |
- 입력 예시

데이터 유입이 되지않아 확인 부탁드립니다.

- 기대 출력

보내주신 데이터의 경로가 정의되지않아 유입이 되지않고 있습니다. 정의 후 사용 부탁드립니다.

- 성공 기준
    - 기준 1: DB 및 로그 기반을 통한 답변 제시
    - 기준 2: DB 및 로그 기반을 통해 답변을 찾지 못할 경우 담당자 확인 필요하다는 안내 제시
